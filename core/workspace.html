workspace — Open documents and manage it¶

Terminology:

Workspace - main working area, where documents are placed.

Document - widget on workspace. Here is examples of documents:

  • Textual editor
  • QtDesigner (probably in the future)
  • QtAssistant (probably in the future)

Document classes are herited from mks.core.abstractdocument.AbstractDocument

mks.core.workspace.Workspace

class mks.core.workspace.Workspace(mainWindow)¶

Bases: PyQt4.QtGui.QStackedWidget

Class manages set of opened documents, allows to open new file

instance is accessible as:

from mks.core.core import core
core.workspace()
documentOpened¶

documentOpened(mks.core.abstractdocument.AbstractDocument)

Signal emitted, when document has been created, i.e. textual file opened, or some other document added to workspace

documentClosed¶

documentClosed(mks.core.abstractdocument.AbstractDocument)

Signal emitted, when document was closed

currentDocumentChanged¶

currentDocumentChanged(mks.core.abstractdocument.AbstractDocument old, mks.core.abstractdocument.AbstractDocument current)

Signal emitted, when current document changed, i.e. user selected another document, new document opened, current closed

del_()¶

Terminate workspace. Called by the core to clear actions

documentForPath(filePath)¶

Find document by it’s file path. Raises ValueError, if document hasn’t been found

setTextEditorClass(newEditorClass)¶

Set text editor, which is used for open textual documents. New editor would be used for newly opened textual documents.

newEditorClass is class, herited from mks.core.abstractdocument.AbstractDocument

textEditorClass()¶

Get text editor, which is used for open textual documents. Text editor is class, herited from mks.core.abstractdocument.AbstractDocument

eventFilter(obj, event)¶

NOT AN API function

Handler for QObject events of children

setCurrentDocument(document)¶

Select active (focused and visible) document form list of opened documents

currentDocument()¶

Returns currently active (focused) document. None, if no documents are opened

goToLine(filePath, line, column, selectionLength)¶

Open file, activate it, and go to specified line.

selectionLength specifies, how much characters should be selected

closeDocument(document, showDialog=True)¶

Close opened file, remove document from workspace and delete the widget

If showDialog is True and file is modified, dialog will be shown

openFile(filePath)¶

Open named file using suitable plugin, or textual editor, if other suitable editor not found.

Returns document, if opened, None otherwise

Opens modal dialog, if failed to open the file

createEmptyNotSavedDocument(filePath=None)¶

Create empty not saved document. Used on startup, if no file was specified, and after File->New file has been triggered

openedDocuments()¶

Get list of opened documents (mks.core.abstractdocument.AbstractDocument instances)

closeAllDocuments()¶

Close all documents

If there are not saved documents, dialog will be shown.

Handler of File->Close->All

Returns True, if all files had been closed, and False, if save dialog rejected

focusCurrentDocument()¶

Set focus (cursor) to current document.

Used if user has finished work with some dialog, and, probably, want’s to edit text
